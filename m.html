الذكاء الاصطناعي: مسارات بلا حدود

رحلة بصرية تفاعلية، نغوص فيها داخل عقول الخوارزميات (Greedy & A*) باستخدام الرسوم المتحركة والأكواد البرمجية الفخمة.

1. لماذا نحتاج خوارزميات البحث؟

الذكاء الاصطناعي لا يرى العالم كما نراه، هو يحتاج لمعادلات رياضية ليتخذ قرار التحرك من النقطة أ إلى النقطة ب بأقل تكلفة، سواء في خرائط جوجل، الألعاب المعقدة، أو توجيه الروبوتات.

التكلفة الفعلية g(n)

التكلفة المؤكدة والجهد الحقيقي الذي تم بذله للوصول للنقطة الحالية. (الماضي)

الحدس التقديري h(n)

تخمين وذكاء الخوارزمية للمسافة المتبقية، يسمى بالاستدلال (Heuristic). (المستقبل)

التقييم النهائي f(n)

مقياس التفوق، المعادلة التي تحسم أي المسارات هو الأفضل ليتم اختياره فوراً.

2. الخوارزمية الجشعة (Greedy BFS)

الاندفاع السريع

تقرر مسارها اعتماداً كلياً على الحدس التقديري h(n) فقط.

f(n) = h(n)

المعادلة التي تتجاهل تماماً كل التكاليف التي قطعتها مسبقاً.

مخاطر الخوارزمية

سرعتها تأتي بضريبة، فقد تنخدع بمسار يبدو قريباً لكنه طريق مسدود!

المحاكاة الحية: كيف تتحرك الخوارزمية الجشعة؟

النقطة المضيئة تمثل الخوارزمية وهي تختار دائماً الرقم الأقل داخل الدائرة (قيمة h)، وتندفع من S إلى B إلى D ثم G، متجاهلة أي تكلفة في الطريق.

الكود البرمجي لخوارزمية Greedy (لقطة 3D)

greedy_algorithm.py
def greedy_bfs(graph, heuristics, start, goal): open_list = [(start, [start])] closed_list = set() while open_list: # الترتيب يعتمد حصرياً على h(n) التقديرية open_list.sort(key=lambda x: heuristics[x[0]]) current_node, path = open_list.pop(0) if current_node == goal: return path closed_list.add(current_node) for neighbor in graph[current_node]: if neighbor not in closed_list: open_list.append((neighbor, path + [neighbor])) return None

تفكيك الأسطر البرمجية:

التهيئة وقوائم التتبع

نبدأ بتعريف open_list لتخزين العقد المكتشفة، و closed_list لحفظ العقد التي زرناها لضمان عدم الوقوع في دوامة.

الفرز الجشع (الجوهر)

السطر sort(...) يرتب القائمة المفتوحة في كل دورة بناءً على القيمة التقديرية h(n) فقط لدفع العقدة الأقرب للأمام.

السحب واختبار الهدف

عن طريق pop(0) نسحب دائماً أول عقدة. ثم نتأكد if current == goal مما إذا كانت هي النهاية المنشودة لكي نتوقف.

3. خوارزمية البحث النجمي (A* Search)

العبقرية في التوازن

تجمع بين المجهود المبذول فعلياً والمسافة المتبقية لتجنب الأخطاء والمطبات.

f(n) = g(n) + h(n)

المعادلة الذهبية والأكثر استخداماً في عالم الألعاب والملاحة الاحترافية.

ضمان الأمثلية (Optimality)

لن تنخدع بالطرق المسدودة، وتضمن إيجاد أقصر وأفضل مسار بنسبة 100%.

المحاكاة الحية: كيف تتحرك خوارزمية A* بذكاء؟

النقطة النيون تمثل خوارزمية A*. رغم أن العقدة B تمتلك h أقل، إلا أن الخوارزمية ترفضها لأن التكلفة الكلية (f=9) باهظة، فتختار العقدة A (التكلفة f=7) ثم تعبر المسار الأقصر للأبد: S -> A -> C -> G.

الكود البرمجي لخوارزمية A* (لقطة 3D)

a_star_algorithm.py
def a_star_search(graph, heuristics, start, goal): # القائمة تخزن الآن: (العقدة، المسار، g_cost الفعلية) open_list = [(start, [start], 0)] closed_list = set() while open_list: # 🌟 الترتيب الذهبي يعتمد على f(n) = g(n) + h(n) open_list.sort(key=lambda x: x[2] + heuristics[x[0]]) current_node, path, current_g_cost = open_list.pop(0) if current_node == goal: return path closed_list.add(current_node) for neighbor, step_cost in graph[current_node].items(): if neighbor not in closed_list: # حساب التكلفة الجديدة: القديمة + المسافة لهذه الخطوة new_g_cost = current_g_cost + step_cost open_list.append((neighbor, path + [neighbor], new_g_cost)) return None

تفكيك الأسطر البرمجية:

تخزين التكلفة الفعلية (g)

في القائمة المفتوحة نقوم بإضافة البعد الثالث current_g_cost. الخوارزمية الآن تتذكر تماماً كم دفعنا للوصول إلى هنا.

الترتيب الدقيق: f = g + h

السطر x[2] + h(x[0]) يمثل المعادلة الرياضية الذهبية التي تقيم كل مسار بموضوعية تامة وتقدم الأفضل.

حساب المسافات التراكمية

عند المرور على الجيران، يتم استخراج المسافة بين العقدتين وجمعها مع المسافة السابقة لنحصل على new_g_cost لهذا المسار المستقبلي.

جدول المقارنة الهندسي

نظرة شاملة ومقارنة نهائية بين الخوارزميتين في هندسة البرمجيات والذكاء الاصطناعي.

وجه المقارنة Greedy Best-First Search A* Search (A-Star)
المعادلة الأساسية $f(n) = h(n)$ $f(n) = g(n) + h(n)$
الضمانية (Optimality) غير مضمونة مضمونة دائماً
استهلاك الذاكرة (Space) منخفض إلى متوسط (O(bm)) عالي جداً للأسف (O(bd))
